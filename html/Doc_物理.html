  • 解析
  • 计算关联的两个mass之间的位置向量。
  • 每个mass会计算自身的位移，和位置向量之间的偏差向量。
  • 两个偏差向量的求和得到影响向量。
  • 最终向量=影响向量-向量-摩擦力。
  • 将mass的位置和最终向量求和。
  • 核心
  • Single f = Vector4.Dot(vmass1.PositionDelta4f, vector4f) - Vector4.Dot(vmass2.PositionDelta4f, vector4f);
  • Vector4f vector4f2 = vector4f * new Vector4f(f) * friction;
  • 总结：获取当前的移动趋势，并通过摩擦力减少这种趋势。类似于干啥都有阻力，减少事情当前的趋势力度。

  • Spring
  • 简介
  • 冲量直接作用于速度。即: 力 <===> 速度
  • 链接两个质量点，传递弹簧形变力+阻尼力
  • 基础概念
  • 冲量 = 力 × 时间 = 质量 × 速度变化
  • 公式
  • J = F·Δt = m·Δv
  • F=m*a
  • v=v0+a·Δt
  • v=v0+J/m
  • 模型
  • 弹簧模型
  • 胡克定律：F = -k × Δx 。弹簧系数*弹簧形变
  • 弹性力=弹性系数Stiffness* 形变距离*方向
  • 符合胡克定律：
  • F = -k × x
  • F：弹簧的回复力（N，牛顿）
  • k：弹簧的劲度系数（N/m，刚度，越大越硬）
  • x：弹簧的形变量（m，当前长度-原长，正为拉伸，负为压缩）
  • 负号表示力的方向总是与形变量相反，即总是试图恢复到原长。
  • 本实现：冲量 J = F × Δt=m×a× Δt= m × Δv
  • 两个质点的相对速度
  • Δv=J/m1-(-J/m2). 冲量相对两个质点，方向相反
  • Δv=J*(1/m1+1/m2)
  • J=Δv/(1/m1+1/m2) 基于相对速度计算冲量
  • Δv=（弹簧形变）/Δt
  • Δv是两个物体的速度差，这个速度差在Δv刚好产生想要的形变长度
  • 这个速度差需要的冲量为J
  • J/m=v1,J/m1=v2,Δv=v1-v2.
  • 阻尼模型
  • 阻尼力=阻尼系数Damping * 速度差* 方向
  • 速度差：在方向的速度差
  • 抑制弹簧系统的震荡
  • F_damping = -c × v_rel
  • F_damping：阻尼力
  • c：阻尼系数（阻尼越大，系统越快停止振荡）
  • v_rel：两端质量点沿弹簧方向的相对速度
  • 本实现：基于相对速度的冲量修正

  • 3. 脚本功能
  • 3.1 线程管理
  • 通过创建和管理一个后台线程来执行物理模拟，以提高游戏性能。
  • 提供了线程启动（Start）、重启（Restart）和重置（Reset）的方法。
  • 3.2 粒子操作
  • 支持多种粒子操作类型，如位置、旋转、速度、力等。
  • 通过 ParticleAction 结构体封装粒子操作数据，并通过数组进行管理。
  • 3.3 性能分析
  • 通过 ProfilingSegment 类进行性能分析，以优化物理模拟的性能。
  • 3.4 同步与锁
  • 使用同步锁（SyncLock）来确保线程安全。
  • 提供了同步等待（WaitForThreadSync）的方法。
  • 4. 脚本特点
  • 多线程：通过后台线程执行物理模拟，提高游戏性能。
  • 粒子操作：支持多种粒子操作类型，并通过结构体进行封装和管理。
  • 性能分析：提供了性能分析工具，以优化物理模拟的性能。
  • 线程安全：使用同步锁确保线程安全，避免数据竞争和死锁。
