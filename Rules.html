📋 项目维护规则

所有贡献者必读 - 5 分钟快速了解

🔑 5 条核心规则（必记！）

1 修改后必须检查 必做
每次修改文档后，运行自动检查脚本 bash validate_docs.sh，必须通过才能提交。
❌ 违反后果：网站崩溃、链接失效、内容丢失
2 文件名规范：XunPhysics 重要
所有文件必须使用 XunPhysics 命名，禁止使用过时的 QHPhysics 名称。
❌ 违反后果：链接失效、网页无法找到
3 Modal 必须隐藏 必做
所有 HTML 文件的 .note-editor 和 .note-overlay 必须包含 display: none 样式。
❌ 违反后果：UI 对话框挡住页面内容，无法使用
4 Markdown 文件必须存在 必做
HTML 中的所有 fetch() 调用必须指向存在的 markdown 文件，路径必须完全匹配。
❌ 违反后果：页面内容无法加载，显示空白
5 新文件要更新导航 重要
创建新文档后，必须在 Documentation_Hub.html 中添加链接，否则用户无法找到。
❌ 违反后果：新文档无人知晓、文档孤立

⚡ 快速工作流

# 修改文档后的标准流程
$ bash validate_docs.sh # 检查（必做！）
$ git add . && git commit -m "docs: ..." && git push

# 常见命令
$ bash -x validate_docs.sh # 查看详细检查过程
$ grep "错误词" TESTING_GUIDE.md # 查找错误解决方案

🚨 常见错误速查

❌ 错误：链接指向不存在的文件
[AI_Tuning_Getting_Started.html] 链接指向不存在的文件: missing.html

✅ 修复：创建文件或修正链接路径
❌ 错误：Modal 没有隐藏
.note-editor 缺少 display: none

✅ 修复：在 CSS 中添加 display: none;
❌ 错误：Fetch 指向不存在的文件
fetch('architecture.md') 但实际文件是 complete_doc.md

✅ 修复：确保路径和文件名完全匹配
