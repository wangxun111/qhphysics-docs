# 会话沟通摘要 (Session History Summary)
**日期**: 2026-03-10
**主题**: 1档卷线器物理震荡修复 & 项目基础设施建设

## 1. 核心问题回顾
**现象**: 当卷线器摩擦力处于 1档 (Low Friction, ~0.5N) 时，竿尖和线长出现剧烈的高频抖动 (Jitter)。
**原因分析**:
1. **信噪比崩塌**: 1档阈值极低，环境微小力变动占比过大，导致系统在“收线”和“出线”状态间频繁切换。
2. **正反馈回路**: 收线 -> 线变短 -> 拉力暴涨 -> 超过摩擦力 -> 判定出线 -> 线变长 -> 拉力骤降 -> 判定收线。此循环导致震荡。
3. **物理模型缺陷**: 缺乏机械迟滞 (Hysteresis) 和阻尼 (Damping)。

## 2. 解决方案演进 (Reel1stBehaviour.cs)
### v1.0 (失败)
- 仅添加 `Mathf.Lerp` 平滑。
- **结果**: 震荡依旧，只是频率变低，变成大幅度“抽搐”。

### v2.0 (部分有效)
- 区分收线和出线逻辑，降低出线增益。
- **结果**: 出线稳定了，但收线时依然因为拉力突变导致反弹。

### v3.0 (最终稳定版)
**设计理念**: 能量压制与死区控制 (Energy Suppression & Deadzone)。
1. **收线侧 (Reeling In)**:
   - **强制死区 (Deadzone)**: 当负载 > 80% 摩擦力时，强制 `Speed = 0`。彻底切断震荡回路，模拟卸力松脱时的打滑。
   - **安全区限速**: 负载 < 80% 时，速度限制为 20%，防止起步冲击。
2. **出线侧 (Drag Out)**:
   - **动态增益 (Dynamic Gain)**: 
     - 小超载 (<0.5N): `Gain = 0.05` (极粘滞，防抖)。
     - 大超载 (>2.0N): `Gain = 1.0` (全开，防断线)。
3. **代码解耦**:
   - 移除 `AppliedForceEqualizer` 依赖，使用本地 `Mathf.Lerp`。
